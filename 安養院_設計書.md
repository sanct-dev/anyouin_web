## 3. 戦略

### サイトの存在意義

> 現在はなくても機能している。行事のお知らせは寺院頼りと回覧板、あとは地域の重鎮が直接来たり電話する形で伝わっている。若手には住職がLINEでグループに流してもらっている。これはこれで機能させていく。その上で、予備知識的に、特に若手に情報が伝わるように。

### ゴール

**次世代とのつながりをつくる**

- 成功の定義：行事にくる若い世代の女性たちの顔が明るくなっている
- ユーザーの課題：行事に参加するのに情報が少ない
- ユーザーの得るもの：最新情報と行事の背景・意味、初めての人に開かれた雰囲気と記述

### サイトの役割

スマホに最適化された公式Webサイト + SNS

**一言で言うと**  
「情報だけ伝わればいいと思ってきたのに、なぜかほっこりする」

### 守るべきもの（壊してはいけないもの）

- 住職と檀家の直接の連絡関係（直接会いに行く、電話する）
- 便り・回覧板・集まりで情報が伝わる地域のネットワーク
- 電話一本で「ちょっと聞いていいですか」と言える距離感
- 直接行く、くる、会う、足を運ぶ関係性

### やってはいけないこと

- お寺行事の形骸化
- 電子マネーでお賽銭、バーチャル墓参り

## 4. 設計メモ（重要）

### Webサイトを作ることへの危惧

- 「デジタルだよーウエーい」という言説が、アナログ世代をびびらせ萎縮させる
- 長老たちが若い世代に情報の主導権を取られたようで、うっかり皮肉・嫌味が出る → 若い世代が萎縮する
- 運営側が対応を「ネットに出ている」で済ますと直接のやりとりが減り、お寺とのつながりが薄れる

**大事なことは「既存の関係を壊さないこと」。Webはあくまで補助でしかない。Webは潜在的なつながりをゆっくり作るためにある。これは10年後に効いてくる。**

### ウェブが必要になるタイミング

今機能しているのは60代以上の方たちが回覧板を回し、電話して、集まりを作っているから。その方たちがいるうちは機能する。**ウェブが必要になる瞬間は今ではない。でもその瞬間に備えて今作る、というのが判断の根拠。**

### 「人」が見えること

住職がゲーム好きだとか、節分で劇をやるとか、そういう「人間くさい部分」がちらっと見えることで「あ、このお寺は普通に話しかけていい場所なんだ」という安心感が生まれる。

## 5. デザインの方向性

### トーン

> 「おしゃれなカフェのコーヒーより、緑茶でほっとする」感覚

- 荘厳・権威的 → 論外
- おしゃれすぎる → 「自分に関係ない」と感じさせる
- 無理に可愛くする → NG
- 賑やか → NG（善称寺は賑やか、安養院はそうではない）

### キーワード

余白が多い、テキストが完結、柔らかい色合い、子ども・女性に開かれている、初心者に親切、敷居が低い

### 参考サイト

- 善称寺（手書きイラスト・水彩・くすみトーン）
- 太宰府天満宮

### 表層の方向性（3つの軸）

| 軸 | 内容 |
|---|---|
| 色 | 温かみのある色、くすみトーン |
| イラスト | 手書き（クレヨン線画＋水彩） |
| レイアウト | 余白多め |

### 文体

- 専門用語を使わない
- 情報を一覧で見せる（ミニマルは推奨しない）
- 画像の美しさと情報設計の両立
- 全部見せる、スクロールで見える
- 子どもたちが愛着を持てる

### DO / DON'T

| DO | DON'T |
|---|---|
| 情報設計、余白、静的サイト（微細なアニメーション） | 権威的、黒、専門用語、都会的、派手なアニメーション |

## 6. メディア戦略

| 優先度 | メディア | 理由 |
|---|---|---|
| 必須 | WEB | 行事の詳細はSNSだけでは見つからない |
| 必須 | 回覧板・お知らせ郵送 | すでに機能している基本インフラ、絶対不可欠 |
| 推奨 | インスタグラム | 即時お知らせの更新と、イベント情報が画像付きでわかる |
| 検討 | X | ペルソナは利用しているが、運営負担・ブランドの一貫性が崩れるリスク |
| 検討 | 寺院リーフレット | 紙は渡される、目に止まる。寺院の顔としての役割 |
| 検討 | 行事のチラシ | すでにお知らせがある。WebにPDFで載せるのはあり |
| やらない | CTAを目的とした販売サイト | お墓のLPは作るが、CTAが目的ではなく利便性として |
| やらない | 新聞広告・タウン誌 | 近隣に届くタイプだが不要 |

### 必須コンテンツ

お知らせ、お寺について、年間行事、法要・葬儀、御祈願・厄除け、アクセス（所在地・連絡先）、よくある質問

### あるといいコンテンツ

- お墓
- 初めての方へ（作法・持ち物）
- 検討：檀家制度について、仏教について、授与品の紹介（オンラインショップはやらない）

### やらないこと

- オンラインショップ（直接会える関係性の維持）
- 予約サイト（電話や直接訪問の関係性の維持）

### 機能要件

- お知らせ更新機能
- Googleマップ埋め込み
- SNSリンク
- スマホ対応

### お問い合わせ

**電話のみ**（フォームは設けない）

### 更新体制

制作会社対応か住職対応か要検討。保守費用として一人体制で維持できる範囲に設計する。

## 10. ユーザーシナリオ

### シナリオ A（須藤春子 ── 行事当日）

- **状況**：雨が降った
- **最初の疑問**：今日の行事が中止かどうか
- **導線**：トップページのお知らせ欄
- **ゴール**：今日決行されることを知る

### シナリオ B（遠くの檀家さん ── お盆前）

- **状況**：お盆が近づいてきた
- **最初の疑問**：お盆は何時に行けばいいか
- **導線**：トップページのお知らせ欄
- **ゴール**：何時に行けばいいかがわかった。連絡する手間が省けた

### シナリオ C（親族 ── 初めて参加）

- **状況**：法事の場所がLINEで送られてきた
- **最初の疑問**：場所・駐車場・コンビニ・トイレ・持ち物・お布施の相場
- **導線**：トップページ → アクセス → 初めての方へ
- **ゴール**：安心して辿り着ける。信頼感が増す

### シナリオ MP（メインペルソナ ── 行事の司会を頼まれた）

- **状況**：育成会から節分の司会を依頼された
- **最初の疑問**：行事の内容
- **導線**：トップページ → ナビゲーションで行事 → 年中行事
- **ゴール**：安心して行事を引き受けられる。予備知識が得られる。開かれている感じがする

### シナリオ SP（サブ ── 親族が亡くなった）

- **状況**：お父さんが倒れた
- **最初の疑問**：お葬式について
- **導線**：トップページ → お葬式
- **ゴール**：安養院に電話して相談する

### シナリオ その他（お墓を探している）

- **状況**：お墓のことを探している
- **最初の疑問**：お墓を引き受けてくれるか、いくらか
- **導線**：トップページ → お墓
- **ゴール**：お墓の検討材料になる
